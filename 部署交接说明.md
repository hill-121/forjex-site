# 🔧 网站部署交接说明

> 给负责部署的同事：请按以下步骤将网站上线到 Netlify。

## 📦 需要部署的文件

所有文件在 `website/` 文件夹中，共 10 个核心文件：

```
index.html                 ← 首页
products-training.html     ← 教学培训设备页
products-flowmeter.html    ← 流量计及仪表页
products-automation.html   ← 自动化设备页
about.html                 ← 关于我们页
style.css                  ← 样式（所有页面共用）
script.js                  ← 交互逻辑（所有页面共用）
robots.txt                 ← 搜索引擎爬虫规则
sitemap.xml                ← 站点地图
netlify.toml               ← Netlify 配置（已配好）
```

> ⚠️ `.md`、`.json` 文件不需要上传，放着也不影响。

---

## 🚀 部署步骤（约 5 分钟）

### 第一步：注册 Netlify
1. 打开 https://app.netlify.com/signup
2. 用 **GitHub 账号** 登录（推荐），或用邮箱注册

### 第二步：部署网站
1. 登录后点击 **"Add new site"**
2. 选择 **"Deploy manually"**
3. **把上面 10 个文件直接拖拽到页面虚线区域**
4. 等待上传完成（通常 10-30 秒）
5. Netlify 会分配一个临时域名，类似：
   `https://random-name-abc123.netlify.app`
6. ✅ **此时网站已经可访问了！**

### 第三步：记录域名
把分配到的临时域名保存好，后续绑定自定义域名要用。

## 🌐 绑定自定义域名（可选）

### 方案 A：免费域名 is-a.dev
1. 需要 GitHub 账号
2. Fork https://github.com/is-a-dev/register
3. 在 `domains/` 文件夹创建 `zktech.json`：
```json
{
  "owner": { "username": "你的GitHub用户名" },
  "records": { "CNAME": "你的Netlify临时域名.netlify.app" }
}
```
4. 提交 Pull Request，等 1-3 天审核
5. 审核通过后，在 Netlify 后台绑定域名

### 方案 B：购买自有域名
- 阿里云/腾讯云购买 `.com` 或 `.cn` 域名
- Netlify 后台 → Site settings → Domain management → Add domain
- 按提示配置 DNS

## 📌 常见问题

**Q: 上传后页面样式丢失？**
A: 确认 `style.css` 和 `script.js` 与 `index.html` 在同一目录下。

**Q: 子页面链接 404？**
A: 确认 `netlify.toml` 文件已上传（它配置了 SPA 回退规则）。

**Q: 想修改产品内容？**
A: 直接编辑对应的 `products-*.html` 文件，重新上传即可。

**Q: 想更新网站怎么办？**
A: 修改文件后，再次到 Netlify 后台 → Deploys → 拖拽上传新版本。
